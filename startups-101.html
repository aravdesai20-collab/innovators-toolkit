Startups 101

Learn the essentials of building a startup

A startup is basically a new business that tries to solve a problem and grow fast. The key is building something people actually want. Think apps for students, services for your local area, or tools that make everyday tasks easier.

MVPs

Build the simplest version that solves the core problem. Learn fast.

Example: A landing page that collects emails before building the full product.
MVP types: Landing pages, demos, manual processes, single-feature apps. Launch in days, not months.

Funding Basics

Not every startup needs funding. Bootstrap when possible.

Bootstrap: Full control, slower growth
Investors: Faster growth, less control
Funding stages: Pre-seed (friends/family), Seed ($100K-$2M), Series A ($2M+). Most never raise.

Problem-Solution Fit

Make sure your solution actually solves a real problem.

Ask yourself: Would users switch from their current solution?
Fit signs: Users pay, refer others, use daily. If not, pivot or find new problem.

Quick Tips

Validate before building
How: Talk to 10+ users, ask about willingness to pay, check competitors.
Start with an MVP
How: Build the simplest version that solves the core problem. Launch in days.
Talk to real users
How: Go where they hang out. Ask about pain points. Listen actively.
Focus on one problem
How: Pick the biggest pain point. Solve it well. Then expand.
