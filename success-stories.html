StudySpace Finder

Web App

A web app that shows available study spaces on campus using real-time WiFi data

Built by: College sophomore
Timeline: 3 weeks MVP, launched to 500+ users in first month
Tools used: React, Firebase, Python (for data), Vercel hosting
Key Challenge: Getting access to real-time building data - solved by scraping public WiFi usage APIs
Biggest Learning: "Talk to users first! I initially built features no one wanted."
Status: Active, ~2,000 monthly users, considering monetization
üí° Key Insights:
  • Used open data from campus WiFi systems
  • MVP was just a simple map showing busy/quiet areas
  • No funding needed - built solo, used free tiers
  • Got users by sharing in campus Facebook groups

AI Study Buddy

AI Tool

A Chrome extension that uses AI to help students summarize notes and create study guides

Built by: High school senior
Timeline: 2 weeks for MVP, iterated based on user feedback for 1 month
Tools used: OpenAI API, Chrome Extension API, JavaScript, GitHub Pages
Key Challenge: Managing API costs - solved by caching responses and optimizing prompts
Biggest Learning: "AI is powerful but you need good prompt engineering. Spent days perfecting prompts."
Status: Active, 1,500+ installs, considering freemium model
üí° Key Insights:
  • Started with free OpenAI credits ($5/month)
  • Chrome Web Store is great for distribution
  • User feedback was critical - rebuilt UI 3 times
  • No backend needed - everything runs client-side

Textbook Exchange Platform

Marketplace

A platform for students to buy and sell used textbooks directly from each other

Built by: Two college freshmen
Timeline: 4 weeks MVP, launched before fall semester, processed $5K in first semester
Tools used: Next.js, Stripe, PostgreSQL, Vercel, Supabase
Key Challenge: Trust between buyers/sellers - solved with ratings and escrow system
Biggest Learning: "Payment processing is complex! Stripe made it much easier than we expected."
Status: Active, expanding to 3 campuses, considering mobile app
üí° Key Insights:
  • Started campus-specific (easier to launch)
  • Used Stripe Connect for marketplace payments
  • No inventory needed - students list their own books
  • Made $500 commission in first semester (10% fee)

Local Fitness Tracker

Mobile App

An app that connects local runners and creates community fitness challenges

Built by: College junior (CS major)
Timeline: 6 weeks MVP, launched in local running community, 800+ users
Tools used: React Native, Firebase, Apple Health API, Expo
Key Challenge: Getting users to download app - solved by partnering with local running clubs
Biggest Learning: "Community features are hard but drive engagement. Most users joined challenges."
Status: Active, exploring partnerships with local gyms
üí° Key Insights:
  • Built mobile-first (fitness apps need mobile)
  • Used Expo for faster development
  • Integrated with Apple Health for automatic tracking
  • Started local, expanding city by city

Campus Food Deals Aggregator

Web App

Aggregates food deals and discounts from restaurants near campus

Built by: College sophomore (non-technical, learned to code)
Timeline: 2 months learning + 3 weeks building, launched to campus, 1,200+ users
Tools used: HTML/CSS/JS (learned basics), Google Sheets API, GitHub Pages
Key Challenge: Manual data entry - solved by asking restaurants to submit their own deals
Biggest Learning: "You don't need to be a CS major. YouTube tutorials and persistence got me there."
Status: Active, partnered with 15+ restaurants, considering expansion
üí° Key Insights:
  • Started with zero coding experience
  • Used Google Sheets as database (simplest solution)
  • Email list was key for user retention
  • Restaurants loved the free marketing

Tutoring Match Platform

Platform

Connects students who need help with other students who can tutor

Built by: Two high school juniors
Timeline: 5 weeks MVP, launched at school, matched 50+ students
Tools used: Bubble (no-code), Stripe, Google Forms (for matching)
Key Challenge: Safety and verification - solved with school email verification
Biggest Learning: "No-code tools are powerful! We built this without writing code."
Status: Active at 2 schools, considering expansion
üí° Key Insights:
  • Used no-code platform (Bubble.io)
  • School partnership was key for trust
  • Took 10% commission on sessions
  • Started with just 5 subjects, expanded based on demand
